        # PostCSS 快速入门

        PostCSS 是一个允许使用 JS 插件转换样式的工具。 这些插件可以检查（lint）你的 CSS，支持 CSS Variables 和 Mixins， 编译尚未被浏览器广泛支持的先进的 CSS 语法，内联图片，以及其它很多优秀的功能。

        PostCSS 在工业界被广泛地应用，其中不乏很多有名的行业领导者，如：维基百科，Twitter，阿里巴巴， JetBrains。PostCSS 的 Autoprefixer (opens new window) 插件是最流行的 CSS 处理工具之一。

        Twitter 账号: @postcss (opens new window)。
        支持 / 讨论: Gitter (opens new window)。

        # 插件

        截止到目前，PostCSS 有 200 多个插件。你可以在 插件列表 (opens new window) 或 搜索目录 (opens new window) 找到它们。 下方的列表是我们最喜欢的插件 - 它们很好地演示了我们可以用 PostCSS 做些什么。

        如果你有任何新的想法，开发 PostCSS 插件 (opens new window) 非常简单易上手。

        # 解决全局 CSS 的问题

        • postcss-use (opens new window) 允许你在 CSS 里明确地设置 PostCSS 插件，并且只在当前文件执行它们。
        • postcss-modules (opens new window) 和 react-css-modules (opens new window) 可以自动以组件为单位隔绝 CSS 选择器。
        • postcss-autoreset (opens new window) 是全局样式重置的又一个选择，它更适用于分离的组件。
        • postcss-initial (opens new window) 添加了 all: initial 的支持，重置了所有继承的样式。
        • cq-prolyfill (opens new window) 添加了容器查询的支持，允许添加响应于父元素宽度的样式.

        # 提前使用先进的 CSS 特性

        • autoprefixer (opens new window) 添加了 vendor 浏览器前缀，它使用 Can I Use 上面的数据。
        • postcss-cssnext (opens new window) 允许你使用未来的 CSS 特性（包括 autoprefixer）。
        • postcss-image-set-polyfill (opens new window) 为所有浏览器模拟了 image-set (opens new window) 函数逻辑。

        # 更佳的 CSS 可读性

        • precss (opens new window) 囊括了许多插件来支持类似 Sass 的特性，比如 CSS 变量，套嵌，mixins 等。
        • postcss-sorting (opens new window) 给规则的内容以及@规则排序。
        • postcss-utilities (opens new window) 囊括了最常用的简写方式和书写帮助。
        • short (opens new window) 添加并拓展了大量的缩写属性。

        # 图片和字体

        • postcss-assets (opens new window) 可以插入图片尺寸和内联文件。
        • postcss-sprites (opens new window) 能生成雪碧图。
        • font-magician (opens new window) 生成所有在 CSS 里需要的 @font-face 规则。
        • postcss-inline-svg (opens new window) 允许你内联 SVG 并定制它的样式。
        • postcss-write-svg (opens new window) 允许你在 CSS 里写简单的 SVG。

        # 提示器（Linters）

        • stylelint (opens new window) 是一个模块化的样式提示器。
        • stylefmt (opens new window) 是一个能根据 stylelint 规则自动优化 CSS 格式的工具。
        • doiuse (opens new window) 提示 CSS 的浏览器支持性，使用的数据来自于 Can I Use。
        • colorguard (opens new window) 帮助你保持一个始终如一的调色板。

        # 其它

        • postcss-rtl (opens new window) 在单个 CSS 文件里组合了两个方向（左到右，右到左）的样式。
        • cssnano (opens new window) 是一个模块化的 CSS 压缩器。
        • lost (opens new window) 是一个功能强大的 calc() 栅格系统。
        • rtlcss (opens new window) 镜像翻转 CSS 样式，适用于 right-to-left 的应用场景。

        # 语法

        PostCSS 可以转化样式到任意语法，不仅仅是 CSS。 如果还没有支持你最喜欢的语法，你可以编写一个解释器以及（或者）一个 stringifier 来拓展 PostCSS。

        • sugarss (opens new window) 是一个以缩进为基础的语法，类似于 Sass 和 Stylus。
        • postcss-html (opens new window) 允许你在 HTML / Markdown (opens new window) / Vue component (opens new window) 里编写样式。
        • postcss-scss (opens new window) 允许你使用 SCSS (但并没有将 SCSS 编译到 CSS)。
        • postcss-sass (opens new window) 允许你使用 Sass (但并没有将 Sass 编译到 CSS)。
        • postcss-less (opens new window) 允许你使用 Less (但并没有将 LESS 编译到 CSS)。
        • postcss-less-engine (opens new window) 允许你使用 Less (并且使用真正的 Less.js 把 LESS 编译到 CSS)。
        • postcss-js (opens new window) 允许你在 JS 里编写样式，或者转换成 React 的内联样式／Radium／JSS。
        • postcss-safe-parser (opens new window) 查找并修复 CSS 语法错误。
        • midas (opens new window) 将 CSS 字符串转化成高亮的 HTML。

        # 使用方法

        你可以通过简单的两步便开始使用 PostCSS：

        1. 在你的构建工具中查找并添加 PostCSS 拓展。
        2. 选择插件 (opens new window)并将它们添加到你的 PostCSS 处理队列中。

        # Webpack

        在 webpack.config.js 里使用 postcss-loader (opens new window) :

        module.exports = {
    module: {
        rules: [
            {
                test: /\.css$/,
                exclude: /node_modules/,
                use: [
                    {
                        loader: 'style-loader',
                    },
                    {
                        loader: 'css-loader',
                        options: {
                            importLoaders: 1,
                        }
                    },
                    {
                        loader: 'postcss-loader'
                    }
                ]
            }
        ]
    }
}

        然后创建 postcss.config.js:

        module.exports = {
    plugins: [
        require('precss'),
        require('autoprefixer')
    ]
}

        # Gulp

        使用 gulp-postcss (opens new window) 和 gulp-sourcemaps (opens new window).

        gulp.task('css', function () {
    var postcss    = require('gulp-postcss');
    var sourcemaps = require('gulp-sourcemaps');

    return gulp.src('src/**/*.css')
        .pipe( sourcemaps.init() )
        .pipe( postcss([ require('precss'), require('autoprefixer') ]) )
        .pipe( sourcemaps.write('.') )
        .pipe( gulp.dest('build/') );
});

        # npm run / CLI

        如果需要在你的命令行界面或 npm 脚本里使用 PostCSS，你可以使用 postcss-cli (opens new window)。

        postcss --use autoprefixer -c options.json -o main.css css/*.css

        # 浏览器

        如果你想编译浏览器里的 CSS 字符串（例如像 CodePen 一样的在线编辑器）， 只需使用 Browserify (opens new window) 或 webpack (opens new window)。它们会把 PostCSS 和插件文件打包进一个独立文件。

        如果想要在 React 内联样式／JSS／Radium／其它 CSS-in-JS (opens new window) 里使用 PostCSS， 你可以用 postcss-js (opens new window) 然后转换样式对象。

        var postcss  = require('postcss-js');
var prefixer = postcss.sync([ require('autoprefixer') ]);

prefixer({ display: 'flex' }); //=> { display: ['-webkit-box', '-webkit-flex', '-ms-flexbox', 'flex'] }

        # JS API

        对于其它的应用环境，你可以使用 JS API：

        const fs = require('fs');
const postcss = require('postcss');
const precss = require('precss');
const autoprefixer = require('autoprefixer');

fs.readFile('src/app.css', (err, css) => {
    postcss([precss, autoprefixer])
        .process(css, { from: 'src/app.css', to: 'dest/app.css' })
        .then(result => {
            fs.writeFile('dest/app.css', result.css);
            if ( result.map ) fs.writeFile('dest/app.css.map', result.map);
        });
});

        阅读 PostCSS API 文档 (opens new window) 获取更多有关 JS API 的信息.

        所有的 PostCSS 运行器应当通过 PostCSS 运行器指南 (opens new window)。

        # 配置选项

        绝大多数 PostCSS 运行器接受两个参数：

        • 一个包含所需插件的数组
        • 一个配置选项的对象

        常见的选项：

        • syntax: 一个提供了语法解释器和 stringifier 的对象。
        • parser: 一个特殊的语法解释器（例如 SCSS (opens new window)）。
        • stringifier: 一个特殊的语法 output 生成器（例如 Midas (opens new window)）。
        • map: source map 选项 (opens new window).
        • from: input 文件名称（大多数运行器自动设置了这个）。
        • to: output 文件名称（大多数运行器自动设置了这个）。
